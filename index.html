Well before we answer all the above questions, lets have a look at a few basics of computers and the web technology as a whole. To do the web development, you don't really need to be knowing who created it first and all as of now, but if you're interested, you can always Google it up.
So, we all know what the URL Bar of a browser is, I'll be using Firefox to be showing you the demos.

What we type out in the URL(Uniform Resource Locator) bar is called a client query. There are two things here, whats a client and whats a server ? A client is a computer that requests information from another computer. A server is a computer that provides you with the information. So think about it ... Can a client be a server ? Can a server be a client ?

So, what happens when we do a query for Google's homepage. the typed www.google.com gets converted to an IP address, something like 74.125.236.179

This number differs based on your closest Google server and their server load, i.e. simultaneous number of people requesting the same page, so instead of typing in www.google.com, if you type in 74.125.236.179 , it'll still take you to the google homepage. This is because the URL is a human readable format of these numbers and each of these names is called a 'Domain Name'. There is a system of naming called DNS which doesn't allow the same name to be taken over, so you can't register your website over an already registered domain name, like google.com. Getting back to where we were, as you type the domain name, it gets converted to an IP address which is computer understandable, this then asks the root servers (the main servers which keep a list of domain names and their connecting numbers), it receives an IP address corresponding to its query. So the computer then calls this computer, this computer(server) then understands the call and sends back the required files to be shown. The medium which makes this connections possible and content viewable are the internet browsers like Firefox, Safari, Internet Explorer.

For those interested in how this connections take place and processes involved, look at a more advanced thing called layers, the approximate working is denoted by something called an OSI Model. Starting from

So are there any other editors and software for web development ?

Yes these are called source text editors, there are some editors which support many programming and scripting languages and come in three ways. Freeware, Opensource and Proprietary software. What exactly is the difference ?

A freeware is a type of computer software that is given completely for free for lifetime usage. eg.HTMLKit. An Open source software is a freeware but the complete source code which is used to make the software is also provided. eg. Notepad++. A proprietary software mostly WYSIWYG software is something which one has to buy and use. eg. Adobe Dreamweaver, Microsoft Visual Studio Web Essentials.

So whats the big difference between these ? Advantages of a source text editor are

  1. Syntax highlighting.
  2. Templates (present in a few proprietary ones).
  3. Toolbars and Keyboard shortcuts.
  4. Automatic insertion and suggestion of HTML elements.
  5. Auto completion.

An HTML page that is created generally contains a Doctype, which defines the version of HTML used in the document. So how is this working,

  • The doctype line says that the document is written in HTML5
  • The <html> tag says that this is the start of the HTML code and </html> marks the end of the HTML code.
  • The <title> My First Webpage </title> tag puts the content My First Webpage between the open and close tags on the title of the browser, In the above figure, the title is present in the tab that is open.
  • The <h1> shows the heading size.
  • The <p> paragraph tag shows how to specify a paragraph. The open and close tags specify one paragraph, to make a new paragraph, a new paragraph tag has to be created below the first paragraph tag.
  • The <body> tags make sure that the information is put in the body of the browser.

Day 6- The Journey of 'Data'

So as you can imagine from the title, there won't be any 'code' today and it'll be mostly theoretical. Today we'll be discussing about how communication takes place between one computer and another, and how information is sent, in detail. This is the journey of data.


Every communication that happens between two computers happens with the help of 14 different layers with 7 layers layers on each computer. A layer can be compared to a real life situation of a train station. There are 7 layers in the OSI model (Open System Interconnect) model. The layers are in the following order in a computer.


  • Application layer.
  • Presentation layer.
  • Sessions layer.
  • Transport layer.
  • Network layer.
  • Data link layer.
  • Physical layer.

So lets analyze the journey of data from one computer to another. First the application layeracts as the window for application processes and users to access networks, compare it to your browser which is an application, the links you click on your screen are the data. Assume this to be the same as a train waiting at a train station, and the data is the cargo its transporting. The application layer packs the data and sends it to the presentation layer. So similarly, the cargo is packed into a train and the train is sent to the next station.


The Presentation Layer acts as the translator to the network, it converts data from common readable format to a much well known network format for the computers, such as converting ASCII to EBCDIC. This is similar to making sure that the train driver knows the language of the place where he's going. This layer also provides data compression and data encryption. Encryption is the process of converting the data from a readable format to a more secure format using mathematical and logical equations. This was first done by Julius Caesar. Something that is plainly readable is called 'plain text' and something that is converted into secret code is called 'cipher text'. The process of converting plain text to cipher text is called Encryption and its reverse is called Decryption. The process and Encryption and Decryption together is called 'Cryptography' and is a very important factor to keep you safe online. Julius Caesar converted every document that has to be sent across a war to a +3 character set, so A becomes D, B becomes E and so on and X becomes A. The information that was being sent could never be read by the enemies in case the messenger got caught. Smart right ?


The Sessions Layer is like the traffic police, it checks the data, and puts the safe to move or similar to a kind of police clearance. It's like a toll gate and remembers which train has left and which train has come, it doesn't allow other data to come and destroys it. It is a further layer of security. It keeps count of how long you're online without viewing a page and them times out that page for security reasons. This acts as what is mostly known as Firewall.


The Transport Layer ensures that the messages are delivered error free in sequence and with no losses or duplications. It relieves the higher layers discussed from any concern with the data. This is similar to a police check of the cargo on the train when its going. It tells the driver where to go with the cargo and its duty is to send the trains carrying the information in order. This layer can accept large messages but the layers to follow have message size requirements. The transport layer breaks the information into smaller segments and sends the information to the Network Layer.


The Network Layer converts the segments into packets similar to how the cargo on the train is distributed equally into smaller carriage cars. This layer is like a junction and it has all possible connections through which it sends the data through one particular track/path just like how trains can change path of movement at junction until they reach another junction. The source and destination of the data is added here to the message.


bits in the Physical Layer and these bits are sent across hardware medium like Ethernet cables, coaxial cables or optic fibre cables from one computer to another.


All this takes place at one side (i.e. information senders), the same steps take place in the destination computer in the reverse direction from Physical layer to Data link layer all the way up to the Application layer. The information that was requested is then sent back from the destination computer through the same 7 steps as we discussed in application layer to physical layer order (since now it acts as sender) and at the receiving it goes back from bottom layer to top.
